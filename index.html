Frequently Asked Questions

1. What is the difference between var, let, and const?

Var: Var is function scoped that means it can be accessed from anywhere inside the function(it is not confined within a block). Also, var can be both reassigned and redeclared inside the same scope. Var is hoisted when the script is loaded but is not defined, initially it is assigned as undefined so that it can be declared latter.

Let: Let is block scoped that means it can be accessed only from inside the block it is declared. Alike Var, Let is also hoisted but is not defined so it shows a reference error. The value of let variable can be reassigned but can not be redefined. value can be assigned after declaration.

Const: Const is almost same like let but the main two differences are const have to be assigned during declaration and can not be re assigned like let.

2. What is the difference between map(), forEach(), and filter()?

Map(): Map() returns a new array with the transformed values but it does not modify the original array

forEach(): forEach() executes a function for each element but does not return a new array rather it returns undefined.

Filter(): It returns a new array containing only elements that meet any condition applied but does not modify the original array.

3. Explain arrow functions and how they are different from regular functions.

Arrow functions have a shorter syntax than regular functions and inherit this, which make them ideal for callbacks and functional programming. Unlike regular functions, they do not have their own this, arguments, or the ability to be used as constructors with new.

4. How JavaScript Promises work?

JavaScript Promises handle asynchronous operations by representing a value that might be available now, later, or never. A Promise has three states—pending, fulfilled, or rejected—and uses .then(), .catch(), and .finally() to handle success or failure.

5. How closures work in JavaScript?

A closure in JavaScript is when an inner function remembers and accesses variables from its outer function even after the outer function has finished executing. This allows data encapsulation and is commonly used for private variables, callbacks, and function factories.
